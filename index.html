node-powershell with recursive tail calls

I run into this scenario constantly: I want to check some data and depending on the result, continue recursively running my algorithm on the given set of data. In this example I am just counting numbers but usually it takes the form of: for a given et of usernames get each AD User's list of group membership and for each of those groups do a thing

When I first created this demo I was using Map instead of the recursive tail-call. To execute my powershell script x 10 I would map a set of 10 values against my node-powershell function and this would spawn 10 powershell processes on the first click. Twenty after a second click... and so on. It was only later on that I realized this scenario isn't just a joke it comes up constantly when building a user interface that drives poweshell work.
It call goes back to the way we write powershell. We say for each child object of this folder, copy the file to network storage. Or we say split this string by a delimiter character and for each substring call a certain cmdlet. If your powershell scripts are simple then you need to orchestrate them in a sophisticated way to actually do anything.
